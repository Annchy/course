- - - 
* ID:						- уникальный номер дефекта 
* Заголовок (summary):				- короткое описание (необходимо ответить
						  на 3 вопроса: где? что? когда?)
* Серьезность (severity):			- серьезность дефекта (блокирующий/
						  критический/cерьезный/
						  незначительный/тривиальный)
  Приоритет (priority):				- приоритет дефекта (высокий/
						  средний/низкий)
  Версия билда (Version):			- билд приложения, на котором был
						  обнаружен дефект
  Статус (status):				- статус дефекта
  Автор (author):				- создатель баг-репорта
- - -
* Окружение (enviroment):			- информация об окружении (ОС/
						  разрядность/сервис пак/браузер)
- - -
* Шаги воспроизведения (steps to reproduce):	- описание шагов, которые приводят
						  к воспроизведению дефекта
  1. 
  2.
  ...
  n. 
* Ожидаемый результат (expected result):	- ожидаемый (корректный) результат
* Фактический результат (result):		- результат, полученный на самом деле
- - -
  Вложения (attachment):			- файлы с логами/изображения/видео,
						  которые могут помочь воспроизвести
						  дефект или решить его
- - -

* - обязательные атрибуты баг-репорта
